01

What it does

The runner executes Luau in the local Roblox-compatible runtime and returns stdout, stderr, capture files, capture index, compatibility trace, owner-protection status, and Luraph recovery reports when present.

Use executor-client for analysis and capture hooks. Use roblox-client for cleaner script-visible behavior with analysis helpers hidden by default.

02

Network modes

  • offline: blocks runtime HTTP. This is the default.
  • allowlist: permits only hosts listed in Allow hosts.
  • live: permits live HTTP. Use this only when you trust the script.

When an offline run reaches HTTP(S), the console displays the exact host and URL. Allow host and retry reruns with allowlist mode and only that host. Redirects are checked at every hop and cannot escape the approved host.

Executor requests use the Opiumware-compatible Roblox/WinInet default User-Agent, preserve script-supplied headers and cookies, and return non-2xx responses with Success=false. Opiumware-shaped identity headers use a fresh random 96-character value for each runtime process; no hardware identifier is read, forwarded, or retained.

In executor-client, game:HttpGet, game:HttpGetAsync, game:HttpPost, and game:HttpPostAsync use the same consent-aware network policy. These executor aliases remain unavailable in roblox-client.

The runner accepts local .lua, .luau, and .txt files up to 900 KB. Fetch URL retrieves remote Luau without executing it, then exposes the complete captured artifact in Files with a SHA-256 identifier and download action.

03

Run outcomes

  • completed: execution and scheduled work completed normally.
  • virtual_budget: a yielding script reached the deterministic simulation limit. This is a successful bounded run.
  • network_required: execution needs explicit approval for one or more HTTP(S) hosts.
  • wall_timeout: CPU-bound execution exceeded the watchdog.
  • runtime_error: compilation or script execution failed.

API responses include terminationReason and a deduplicated networkRequirements array with host, url, and policy fields.

04

Hook options

  • analysisHooks: exposes capture and executor-analysis hooks when on.
  • captureStringHooks: hooks large string-producing paths such as table.concat, string.gsub, coroutine returns, and buffer output.
  • traceCalls: writes script-visible API calls to the compatibility trace.
  • tracePcallErrors: captures normalized protected-call errors.
  • autorunLoadstring: executes loaded chunks immediately for analysis.

06

Automatic deobfuscator

The deobfuscator workbench combines static recovery with bounded runtime captures. It runs offline, removes its temporary workspace before responding, and exposes source, VM disassembly, semantic IR, CFG, provenance, and report views.

Exact succeeds only for verified source-bearing evidence. Readable may infer structure and generated names, so it is always labeled reconstructed. VM only never emits a source claim. Completed source responses include a 0–100 source-quality score that measures residual VM-shaped names, register tables, aliases, capture wrappers, numeric noise, and state-machine artifacts separately from runtime equivalence.

07

Deobfuscation jobs

Create returns capability URLs for status and the finished response. Poll status once per second until state is completed or failed.

curl -X POST https://YOUR_HOST/jobs/create \
  -H 'content-type: application/json' \
  --data-binary @request.json

curl https://YOUR_HOST/jobs/status/JOB_ID
curl https://YOUR_HOST/jobs/finished/JOB_ID
{
  "source": "print('hello')",
  "mode": "reconstruct",
  "profile": "executor-client",
  "time_limit": "auto"
}

time_limit accepts auto, custom, or unlimited. Custom jobs also send wall_timeout from 1 to 280 seconds. Unlimited removes the analyzer watchdog; hosted jobs remain subject to the platform maximum.

08

Local run

From the project root:

cd web/rbx-runtime-runner
RBX_RUNTIME_BINARY=../../build/rbx_luau_runtime PORT=8794 npm run dev

Then open http://127.0.0.1:8794/.

09

Docker

The production image compiles the Linux runtime from source and runs as a non-root user with a read-only root filesystem and tmpfs artifacts.

cd web/rbx-runtime-runner
docker compose up --build

Open http://127.0.0.1:8795/. Set ALEX_RUNTIME_PORT to choose another host port. Health is available at /api/health.
